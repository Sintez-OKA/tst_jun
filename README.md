# Тестовое задание — Python/Django разработчик (frePPLe)

## Цель
Проверить навыки разработки на Python/Django, работы с PostgreSQL, понимание структур данных, умение работать с CSV, писать REST API, тесты, management-команды и следовать стандартам кода.  
Дополнительно оценивается понимание архитектуры и опыт развертывания систем под **Ubuntu**, включая понимание работы с окружением **frePPLe**.

## Исходные данные
В приложении `tst_jun/` находится файл:

orders_extended.csv

Он содержит тестовые производственные заказы:

| Поле | Описание |
|------|----------|
| order_code | Код заказа |
| product | Наименование продукта |
| quantity | Количество |
| due_date | Дата исполнения |
| priority | Приоритет (1 — критичный, 3 — низкий) |

---

## Что нужно сделать

### 1. **Импорт CSV в базу**
Создать Django model `Order` и реализовать импорт данных через management-команду.


Требования:
- Импорт должен быть **повторно запускаемым** (без дубликатов)
- Реализовать логирование результатов (сколько создано/обновлено)

---

### 2. **REST API**
Реализовать API:

GET /api/orders/ → список заказов
GET /api/orders/<id>/ → конкретный заказ
POST /api/orders/ → создание заказа

который возвращает:
- Общее число заказов
- Количество по продуктам
- % критичных (priority = 1)
- Средний объём заказа
- Ближайший срок исполнения (min due_date)

---

### 4. **Тесты**
Написать тесты (pytest или unittest) на:
- Импорт CSV
- API /api/orders/
- API /api/orders/statistics/

---

### 5. **Docker (желательно)**
При возможности — добавить `docker-compose.yml`, чтобы поднималось:

- Django app
- PostgreSQL

---

### 6. *(Дополнительно, будет плюсом)*
- Набросок deploy-инструкции для Ubuntu server
- Комментарии, где бы выполнялась интеграция с **frePPLe**
- Примеры планирования заказов в формате JSON для передачи в frePPLe
